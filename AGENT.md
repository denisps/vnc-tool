# AGENT Rules

- Write clean code, like the book clean code teaches.
- Don't leave legacy/depricated/orphaned code.
- If tests are testing for depricated implementation, refactor them.
- Always change the version, last octet for bugfix/minor new feature or middle for major API or breaking changes, but not for documentation or test coverage changes.
- Always stage all changes before committing.
- Always create meaningful, descriptive commit messages.
- Always tag the new version with semver
